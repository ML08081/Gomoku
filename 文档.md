# 五子棋游戏服务器 - 开发文档

## 1. 项目概述

### 1.1 项目简介
本项目是一个基于 C++ 的五子棋网络游戏服务器，运行于麒麟 Linux 系统，用于管理用户注册、登录、匹配、游戏逻辑处理等功能。

### 1.2 技术栈
- **语言**: C++11
- **数据库**: MySQL 8.0+
- **网络**: 原生 Socket (跨平台)
- **构建工具**: CMake 3.10+

### 1.3 主要功能
- 用户注册与登录（密码 SHA256 加密）
- 玩家匹配系统
- 五子棋游戏逻辑
- 对局记录与回放
- 玩家战绩统计

## 3. 核心模块详解

### 3.1 数据库模块 (Database)

**功能**: 封装所有数据库操作，提供玩家管理、对局记录、会话管理等接口。

**关键函数**:
| 函数名 | 功能描述 |
|--------|----------|
| `connect()` | 连接数据库 |
| `registerPlayer()` | 注册新玩家 |
| `loginPlayer()` | 玩家登录验证 |
| `updatePlayerStats()` | 更新玩家战绩 |
| `createMatch()` | 创建对局记录 |
| `addMoveRecord()` | 记录落子 |
| `createSession()` | 创建登录会话 |

**数据结构**:
```cpp
struct Player {
    int player_id;
    std::string username;
    std::string password_hash;
    int total_games, wins, losses, draws;
};

struct Match {
    int match_id;
    int black_player_id, white_player_id;
    int winner;  // 1-黑方, 2-白方, 0-平局
    std::string start_time, end_time;
};
```

### 3.2 网络模块 (TcpSocket)

**功能**: 跨平台 Socket 封装，支持 Windows 和 Linux。

**关键特性**:
- 条件编译处理 Winsock 与 POSIX 差异
- 支持非阻塞模式
- 支持地址复用

**主要方法**:
| 方法名 | 功能描述 |
|--------|----------|
| `create()` | 创建 Socket |
| `bind(ip, port)` | 绑定地址端口 |
| `listen()` | 监听连接 |
| `accept()` | 接受连接 |
| `send(data)` | 发送数据 |
| `recv(buffer)` | 接收数据 |

### 3.3 游戏逻辑模块 (GameLogic)

**功能**: 实现五子棋核心游戏逻辑。

**核心算法**:
- 15x15 棋盘
- 四方向胜负判定（水平、垂直、两条对角线）
- 连五获胜

**主要方法**:
| 方法名 | 功能描述 |
|--------|----------|
| `makeMove(x, y, color)` | 落子 |
| `checkWin(x, y)` | 检查胜负 |
| `isValidMove(x, y)` | 验证落子位置 |
| `reset()` | 重置棋盘 |

### 3.4 服务器核心模块 (Server)

**功能**: 管理客户端连接、消息处理、房间匹配。

**状态机**:
```
未连接 → 已连接 → 已登录 → 匹配中 → 对局中
```

**消息协议**:

| 客户端命令 | 格式 | 描述 |
|-----------|------|------|
| LOGIN | `LOGIN <username> <password> <salt>` | 用户登录 |
| REGISTER | `REGISTER <username> <password> <salt>` | 用户注册 |
| MATCH | `MATCH` | 请求匹配 |
| MOVE | `MOVE <x> <y>` | 落子 |
| SURRENDER | `SURRENDER` | 认输 |
| QUIT | `QUIT` | 退出 |
| STATS | `STATS` | 查询战绩 |
| REPLAY | `REPLAY <match_id>` | 回放对局 |

| 服务器响应 | 格式 | 描述 |
|-----------|------|------|
| LOGIN_OK | `LOGIN_OK <session_id> <total> <wins> <losses> <draws>` | 登录成功 |
| REGISTER_OK | `REGISTER_OK` | 注册成功 |
| MATCH_QUEUE | `MATCH_QUEUE` | 进入匹配队列 |
| MATCH_START | `MATCH_START <match_id> <BLACK\|WHITE>` | 匹配成功开始游戏 |
| MOVE | `MOVE <x> <y>` | 对手落子 |
| TURN | `TURN <1\|2>` | 轮到哪方 |
| WIN | `WIN <BLACK\|WHITE\|DRAW>` | 游戏结束 |
| ERROR | `ERROR <message>` | 错误信息 |

## 4. 数据库设计

### 4.1 数据库表结构

**players 表** - 玩家信息
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| player_id | INT | PRIMARY KEY AUTO_INCREMENT | 玩家ID |
| username | VARCHAR(50) | UNIQUE NOT NULL | 用户名 |
| password_hash | CHAR(64) | NOT NULL | 密码哈希 |
| register_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| total_games | INT | DEFAULT 0 | 总对局数 |
| wins | INT | DEFAULT 0 | 胜场数 |
| losses | INT | DEFAULT 0 | 负场数 |
| draws | INT | DEFAULT 0 | 平局数 |

**matches 表** - 对局记录
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| match_id | INT | PRIMARY KEY AUTO_INCREMENT | 对局ID |
| black_player_id | INT | FOREIGN KEY | 黑方玩家ID |
| white_player_id | INT | FOREIGN KEY | 白方玩家ID |
| winner | TINYINT | NULL | 获胜方 |
| start_time | TIMESTAMP | NOT NULL | 开始时间 |
| end_time | TIMESTAMP | NULL | 结束时间 |
| total_moves | INT | DEFAULT 0 | 总步数 |

**moves 表** - 落子记录
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| move_id | BIGINT | PRIMARY KEY AUTO_INCREMENT | 落子ID |
| match_id | INT | FOREIGN KEY | 所属对局 |
| move_number | INT | NOT NULL | 第几步 |
| player_color | TINYINT | NOT NULL | 1-黑, 2-白 |
| pos_x | TINYINT | NOT NULL | X坐标 |
| pos_y | TINYINT | NOT NULL | Y坐标 |

**game_sessions 表** - 在线会话
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| session_id | VARCHAR(64) | PRIMARY KEY | 会话ID |
| player_id | INT | FOREIGN KEY | 玩家ID |
| login_time | TIMESTAMP | NOT NULL | 登录时间 |
| last_active_time | TIMESTAMP | NOT NULL | 最后活跃时间 |
| ip_address | VARCHAR(45) | NULL | 客户端IP |

## 5. 编译与部署

### 5.1 环境要求

**麒麟 Linux**:
- GCC 7.0+ 或 Clang 6.0+
- CMake 3.10+
- MySQL Server 8.0+
- MySQL Connector/C++ 8.0+

**依赖安装**:
```bash
sudo apt-get update
sudo apt-get install g++ cmake mysql-server libmysqlcppconn-dev libssl-dev
```

### 5.2 编译步骤

```bash
mkdir -p build && cd build
cmake ..
make -j4
```

### 5.3 运行服务器

```bash
# 默认配置运行
./gomoku_server

# 指定端口运行
./gomoku_server 8888

# 完整参数
./gomoku_server <port> <db_host> <db_user> <db_password> <db_name>
```

### 5.4 数据库初始化

执行 `数据库代码.txt` 中的 SQL 语句创建数据库和表：

```bash
mysql -u root -p < 数据库代码.txt
```

## 6. 安全机制

### 6.1 密码安全
- 使用 SHA256 哈希算法
- 随机盐值防止彩虹表攻击
- 盐值由客户端生成并随登录请求发送

### 6.2 SQL 注入防护
- 使用参数化查询（PreparedStatement）
- 禁止拼接 SQL 字符串

### 6.3 会话管理
- 随机生成 32 位十六进制 Session ID
- 定期清理过期会话
- 记录客户端 IP 便于追踪

## 9. 注意事项

1. **网络模式**: 虚拟机需设置为桥接模式
2. **防火墙**: 开放端口 8888（麒麟：`ufw allow 8888`）
3. **字符编码**: 使用 UTF-8
4. **MySQL 线程安全**: 确保开启 Serialized 模式（默认）
5. **Windows 注意**: WSAStartup 必须在 socket 调用前执行
